# Utility Skills

Collection of some utility skills.

I created these for my personal use cases, so there will be some assumptions, requirements and specific paths and structures in the skill body you will need to edit to make it your own:

- I use these within an [Obsidian](https://obsidian.md) Vault.  My vault is structured using [PARA](https://fortelabs.com/blog/para/), expect to see some references to that.
- I assume that [Pandoc](https://pandoc.org/) is installed.
- I assume that you have (at the time of writing this) a [Catalyst license](https://help.obsidian.md/catalyst) for Obsidian, and that you have the [Obdidian CLI](https://help.obsidian.md/cli) installed.

## Skills

### [product-grill](product-grill/SKILL.md)

Stress-tests a product idea before committing. Inspired by Matt Pocock's [Grill Me With Docs](https://www.aihero.dev/grill-with-docs) skill, adapted for product managers. Interviews you across six PM lenses (who, evidence, why now, counterfactual, scope, falsifiability) one question at a time, then produces a conviction brief with a problem statement, assumption stack, decision log, and glossary.

### [interactive-explainer](interactive-explainer/SKILL.md)

Inspired by Simon Willison's [interactive explanations pattern](https://simonwillison.net/guides/agentic-engineering-patterns/interactive-explanations/). Builds a self-contained interactive HTML visualisation of a process, algorithm, decision logic, or system. Takes code, documents, or concept descriptions as input and produces something you can click through to understand how it works.

### [linear-walkthrough](linear-walkthrough/SKILL.md)

Inspired by Simon Willison's [linear walkthroughs pattern](https://simonwillison.net/guides/agentic-engineering-patterns/linear-walkthroughs/). Generates a step-by-step narrative walkthrough of source material — code, documents, architecture diagrams, or any combination — explaining logic, purpose, and connections in a readable sequence.

### [markdown-publisher](markdown-publisher/SKILL.md)

Publishes Markdown files from an Obsidian vault using Pandoc.

### [obsidian-cli](obsidian-cli/SKILL.md)

Interacts with an Obsidian vault via the Obsidian CLI.
